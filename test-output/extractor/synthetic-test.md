# PHOBOS Archive Test Document

## Introduction

This is a synthetic test document generated by test-archive.ts.
It contains sample content about the PHOBOS dual-AI system.

## Architecture

SYBIL is the embedding persona in PHOBOS. It runs nomic-embed-text-v1.5
on CPU using llama-server. SAYON is the coordinator and SEREN is the engine.

## Archive Design

The Archive uses separate DuckDB files per knowledge domain. Each domain
has its own HNSW index for cosine similarity search. Full-text search is
provided by the DuckDB FTS extension as an additive layer.

## Retrieval

Chunks are retrieved via hybrid search: semantic (HNSW cosine similarity)
combined with full-text search (BM25). Results are merged and re-ranked
before injection into the Complete Context as XML blocks.

## Conclusion

The PHOBOS Archive enables persistent, multi-domain knowledge retrieval
that survives across sessions and can be queried by both SAYON and SEREN.
